# Crossing — 5 個 USP wedge

> **這是 handbook 真正的獨家**。foundations/ 任何人都能寫，crossing/ 是把 5 條技術路線橫切後的對比視角 —— 別處看不到。

| Wedge | 一句話 |
|---|---|
| [pixel-vs-latent-physics](pixel-vs-latent-physics/overview.md) | 什麼時候該在 pixel 學物理，什麼時候該在 latent 學 |
| [sim-vs-gen-data](sim-vs-gen-data/overview.md) | 機器人資料：合成 sim、生成模型、真實 demo，哪個贏 |
| [controllability-vs-fidelity](controllability-vs-fidelity/overview.md) | 加越多 conditioning 為什麼 fidelity 越爛 — Pareto |
| [conservation-violation-atlas](conservation-violation-atlas/overview.md) | 誰違反哪些守恆律 — 主流方法 × 5 守恆律的失敗地圖 |
| [text-action-trajectory-spectrum](text-action-trajectory-spectrum/overview.md) | Controllability input 的光譜 — 為什麼 text 不夠、action 不夠、需 multi |

## 寫作標準

每個 wedge 需要：
- **明確的 thesis statement**（一行）
- 跨 ≥ 2 條技術路線的比較
- 至少 3 個 anchor 方法的失效實測
- 一張 ASCII / markdown 對比表
- 不寫 paper summary

## 與 foundations/ 的關係

foundations/ 寫「這條路線是什麼」；crossing/ 寫「這條路線跟那條路線在什麼維度衝突 / 互補」。
