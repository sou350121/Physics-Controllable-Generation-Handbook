## Non-critical observations (nice-to-have follow-ups)

These do not block commit but should be tracked.

1. **`streaming` is not a documented `temporal` value in ontology.md v1.** Both `genesis.md` and `mujoco-mjx.md` use `temporal=streaming`. The ontology v1 spec lists `single-frame / autoregressive / joint-rollout / latent-rollout / hierarchical / temporal-transformer-rolling`. `streaming` is intuitively correct for sim-in-loop (per-step forward), but it should either be added to the ontology v1.1 spec or these two files should switch to `autoregressive`. Recommend adding to ontology v1.1 with explicit anchor "diff-sim / sim-in-loop simulators" because re-tagging both files would feel forced. (The v1.1 review file already proposes substantial renaming on this axis — fold this in.)

2. **`v-jepa-2.md` V-JEPA date claim.** §1 references "Bardes et al., 2024-02" but arxiv 2404.08471 was posted in April 2024. The HuggingFace publication of the paper was Feb 15 2024 but arxiv canonical is April. Suggest changing "2024-02-15" to "2024-04" or clarifying "originally announced Feb 2024, arxiv April 2024".

3. **`vbench-physics.md` and `genesis.md` reference `[TBD: verify]` checklists at end of file** — these are healthy and clearly bounded. No action.

4. **`graphcast.md` has 8 `[TBD: verify]` markers in §7/§8** — highest TBD density of the batch, but all are in non-critical positions (specific issue numbers, ECMWF operational status as of 2026-05). Acceptable; recommend a single pass to close these out within 1-2 weeks.

5. **`dreamer-v4.md` §7 ref 1** uses arxiv 2301.04104 v3 + Nature 2025 — slight stylistic inconsistency vs other files that cite a single canonical. Not a defect.

6. **`cosmos-wfm.md` §7 [TBD]** is a single bracketed note about Cosmos-Drive-Dreams arxiv ID being unconfirmed. Fine for a v1 release; remove once verified.

7. **`sora.md` $15M/day claim flagged in §8.7 as single-sourced.** Author was correctly skeptical and labelled it. Best-practice example for the rest of the corpus.

8. **`gaia-2.md` has 6 [TBD] markers** but each is paired with explicit "what to verify" notes. Same audit philosophy as graphcast.md — acceptable.

9. **`mujoco-mjx.md` §8.9** explicitly labelled `[TBD: verify]` for Apple Silicon Metal perf — appropriate.

10. **Cross-handbook references** to sister VLA-Handbook and Spatial-Intelligence-Handbook appear in `v-jepa-2.md` §6 and `gaia-2.md` §6 — pattern is consistent with AGENTS.md's "與三個 sister handbook 的分工" guidance.
